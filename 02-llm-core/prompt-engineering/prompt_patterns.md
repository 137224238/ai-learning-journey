## 1. 基础模式

### 1.1 角色扮演（Role Prompting）
```
你是一位 [具体角色]，拥有 [具体专业领域] 的深厚经验。
你的职责是 [具体任务]。
你的风格是 [风格描述：严谨/简洁/友好...]。

用户问题：{question}
```

### 1.2 结构化输出（Structured Output）
```
请分析以下内容，并以 JSON 格式返回：

内容：{content}

要求的 JSON 结构：
{
  "summary": "一句话摘要",
  "key_points": ["要点1", "要点2"],
  "sentiment": "positive|neutral|negative",
  "confidence": 0.0-1.0
}

仅返回 JSON，不要其他内容。
```

### 1.3 少样本示例（Few-Shot）
```
请判断以下文本的情感倾向。

示例：
输入：这个产品质量太差了！
输出：负面

输入：还可以，中规中矩。
输出：中性

输入：完全超出了我的预期，太棒了！
输出：正面

现在请判断：
输入：{text}
输出：
```

## 2. 推理增强模式

### 2.1 Chain of Thought（思维链）
```
请一步一步地思考以下问题：

问题：{question}

请按以下格式回答：
思考过程：
1. 首先...
2. 然后...
3. 因此...

最终答案：
```

### 2.2 Tree of Thoughts（思维树）
```
对于以下问题，请探索 3 种不同的解决思路，然后选择最优的：

问题：{question}

思路 A：[描述方案A]
优点：...
缺点：...

思路 B：[描述方案B]
优点：...
缺点：...

思路 C：[描述方案C]
优点：...
缺点：...

最优选择及理由：
```

### 2.3 Self-Consistency（自一致性）
```
请用 3 种独立的方法解决以下问题，然后综合得出最终答案：

问题：{question}

方法 1：
方法 2：
方法 3：

综合结论（多数答案或最合理的）：
```

### 2.4 ReAct 模式（推理 + 行动）
```
你能使用以下工具：
- search(query): 搜索信息
- calculate(expression): 计算数学表达式
- get_date(): 获取当前日期

请按照 Thought → Action → Observation 循环来解决问题。

问题：{question}

Thought: 我需要...
Action: search("...")
Observation: [工具返回结果]
Thought: 根据结果，我...
Action: ...
Observation: ...
...
Final Answer: 
```

## 3. RAG 相关模式

### 3.1 基础 RAG 生成模板
```
你是一个专业的问答助手。请基于以下参考文档回答用户的问题。

参考文档：
{context}

用户问题：{question}

回答要求：
1. 只基于参考文档中的信息回答
2. 如果文档中没有相关信息，请明确说明
3. 引用信息时请注明来源（如"根据文档第X段..."）
4. 回答要简洁准确

回答：
```

### 3.2 HyDE（假设文档嵌入）
```
请根据以下问题，生成一个假设性的、理想的回答文档片段。
注意：这个文档片段将用于向量检索，所以要包含可能相关的关键术语。

问题：{question}

假设性文档片段（150字以内）：
```

### 3.3 查询扩展（Query Expansion）
```
请将以下用户查询改写为 3 个不同角度的搜索查询，以提高检索覆盖率：

原始查询：{query}

改写要求：
1. 保持原始意图
2. 使用不同的措辞或角度
3. 可以更具体或更概括

改写版本：
1. 
2. 
3. 
```

### 3.4 文档相关性判断
```
请判断以下文档片段是否与用户问题相关。

用户问题：{question}

文档片段：
{document}

请回答（只需回答 "相关" 或 "不相关"，以及一句话理由）：
```

### 3.5 引用溯源（Citation）
```
请基于以下文档回答问题，回答中必须注明每个观点的来源文档编号。

文档 [1]：{doc1}
文档 [2]：{doc2}
文档 [3]：{doc3}

问题：{question}

回答（格式：观点[来源编号]）：
```

## 4. 代码生成模式

### 4.1 代码生成基础模板
```
请使用 Python 实现以下功能：

功能描述：{description}

要求：
- 包含详细的注释
- 添加类型注解
- 包含错误处理
- 提供使用示例

实现代码：
```

### 4.2 代码审查模板
```
请对以下代码进行 Code Review：

```python
{code}
```

请从以下维度评估：
1. **正确性**：逻辑是否正确？有没有 bug？
2. **性能**：有没有明显的性能问题？
3. **可读性**：代码是否清晰易懂？
4. **最佳实践**：是否遵循 Python 最佳实践？
5. **安全性**：有没有安全隐患？

以 JSON 格式输出：
{
  "score": 1-10,
  "issues": [{"severity": "high|medium|low", "line": N, "description": "..."}],
  "suggestions": ["建议1", "建议2"],
  "summary": "总体评价"
}
```

### 4.3 测试生成模板
```
请为以下函数生成全面的单元测试：

```python
{function_code}
```

测试要求：
- 覆盖正常情况
- 覆盖边界条件
- 覆盖异常情况
- 使用 pytest 框架

测试代码：
```

## 5. Agent 指令模式

### 5.1 系统提示词（System Prompt）基础框架
```
# 角色
你是 [助手名称]，一个专门用于 [具体用途] 的 AI 助手。

# 能力
你可以使用以下工具：
{tool_descriptions}

# 工作原则
1. 每次只执行一个工具调用
2. 在行动前先思考
3. 如果不确定，询问用户而不是猜测
4. 完成任务后，清楚地报告结果

# 限制
- 不要 [限制1]
- 不要 [限制2]

# 输出格式
{output_format}
```

### 5.2 工具调用规范
```
你有以下工具可以使用，调用时请严格遵守格式：

工具列表：
{tools_json}

调用格式：
<tool_call>
{
  "name": "工具名",
  "arguments": {
    "参数名": "参数值"
  }
}
</tool_call>

注意：
- 每次只调用一个工具
- 等待工具返回结果后再继续
- 不要捏造工具返回结果
```

## 6. 评估与验证模式

### 6.1 LLM-as-Judge（LLM 评估）
```
请评估以下 AI 回答的质量：

问题：{question}
参考答案：{reference}
AI 回答：{answer}

评估维度（各 1-5 分）：
1. 准确性：回答是否正确？
2. 完整性：是否覆盖了关键点？
3. 简洁性：是否简洁不冗余？
4. 有用性：是否对用户有实际帮助？

输出 JSON：
{
  "scores": {"accuracy": N, "completeness": N, "conciseness": N, "helpfulness": N},
  "overall": N,
  "feedback": "具体反馈"
}
```

### 6.2 幻觉检测
```
请检查以下 AI 回答中是否存在幻觉（即模型捏造的信息）：

参考文档：{context}
AI 回答：{answer}

对于回答中的每个关键声明，判断是否有文档支撑：

输出格式：
[
  {"claim": "声明1", "supported": true/false, "evidence": "依据或'无依据'"},
  ...
]
```

## 使用技巧

1. **明确角色**：给模型一个具体的角色可以显著提升输出质量
2. **控制输出格式**：指定 JSON 格式可以方便程序化处理
3. **提供示例**：3-5 个示例通常比大量说明更有效
4. **设置约束**：明确说"只做X，不做Y"
5. **迭代优化**：记录每次调整和效果，找到最优 prompt
